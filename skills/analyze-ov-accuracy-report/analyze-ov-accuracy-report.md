# Analyze the OpenVINO Accuracy Report
1. The overall accuracy data is stored in first sheet of the report (spreadsheet).In this sheet
    a. the "topology" column contains the name of the model
    b. "device" column is for HW platform
    c. "precision" for model precision validated.
    d. there are several columns with "metric"s, such as "metric status WW08_N_G", which contains "passed" or "downgrade" values for each metric. "passed" means pass, "downgrade" means fail.

2. filter device with CPU related hardware platform
3. check all failure cases by value "downgrade" in the "metric" columns, and analyze the failure cases by topology, precision and metric name.
4. summarize the failure cases and identify any patterns or common issues that may be causing the failures
5. summarize and compare metric by metric. for example, new downgrade introduced compared to previous metric. such as "metric status WW08_N_G" vs "metric status WW07_N_G". This can help identify any regressions in the model's performance.
6. if you see some specific topology has more failure cases than others, you can further analyze the failure cases for that topology to identify any specific issues that may be causing the failures. 
    a. You can cross check other device, such as GPU.
        i. if it is CPU specific issue:
            check corresponding "metric value" and compare with previous metric value, such as "metric value WW08_N_G" vs "metric value WW07_N_G". This can help identify any regressions in the model's performance. and compare with GPU metric value to check whether it's CPU specific issue and whether the value is similar or not.
    b. you can check whether it's related to specific precision, such as FP16 or INT8.
